문제 정의

기존 AI 글쓰기 도구는 '정답'과 '교정'에 집중하지만, 정작 창의적인 글쓰기(소설, 에세이)에서는 '영감'이 더욱 중요합니다. 본 프로젝트는 '글쓰기가 막히는 경험'을 해결하기 위해, AI의 예측 불가능성을 '유해한 환각'이 아닌 **'의도된 창의적 변주(Variation)'**로 제어할 수 있다면, 작가에게 강력한 영감을 주는 도구가 될 것이라는 가설에서 출발했습니다. 사용자가 직접 AI의 영감을 '조율'하는 새로운 작문 보조 도구를 개발하는 것을 목표로 했습니다.

아키텍처/설계

LLaMA 3 8B 모델을 Ollama를 통해 100% 로컬 환경에서 구동하여 프라이버시를 확보했습니다. 핵심은 AI의 출력을 제어하는 **'모듈형 RAG 아키텍처'**입니다. 사용자는 UI에서 토글 스위치를 통해 ①글의 설정/캐릭터(로어북), ②문체 제안(스타일), ③창의적 불협화음(와일드카드) RAG 모듈을 개별적으로 껐다 켤 수 있습니다. 이 'AI 믹서' UI를 통해 사용자가 AI의 제안을 직접 '조율'하도록 설계했습니다.

핵심 기능 1: 모듈형 RAG 컨트롤 패널

React와 TypeScript를 사용해 'AI 믹서' UI를 구현했습니다. 사용자가 [글의 설정], [캐릭터], [와일드카드]의 토글을 ON/OFF 하면, 이 상태값이 백엔드(Node.js)로 전달됩니다. 백엔드는 활성화된 RAG 소스(로어북, 와일드카드 등)에서만 정보를 검색하고, 이를 조합하여 최종 프롬프트를 동적으로 생성한 뒤 Ollama API로 전송합니다. 이를 통해 사용자가 실시간으로 AI의 '참조 데이터'를 제어할 수 있게 했습니다.

핵심 기능 2: '창의적 불협화음' 주입

AI의 '유해한 환각'을 억제하는 대신, '의도된 창의성'을 주입하는 '와일드카드 RAG'를 구현했습니다. `wildcards.txt`에는 현재 문맥과 의도적으로 상관없는 낯선 감각, 개념(예: '축축한', '고요한 심해')을 저장했습니다. 사용자가 이 기능을 활성화하면, LLM은 이 '낯선 재료'를 기존 문맥과 창의적으로 연결해야 하는 미션을 받아, 사용자의 막힌 생각의 '물꼬를 트는' 역할을 수행합니다.

나의 역할 및 기여

  • 프로젝트 기획 및 '모듈형 RAG 믹서' 아키텍처 전체 설계
  • Ollama 기반 LLaMA 3 8B 로컬 구동 환경 구축 및 API 연동
  • 3종 RAG(로어북, 스타일, 와일드카드) 데이터베이스 설계 및 백엔드 로직 구현
  • React 및 TypeScript 기반 토글 UI(컨트롤 패널) 프론트엔드 전체 개발

결과 및 회고

  • AI의 제안을 '교정'뿐만이 아닌 '영감'의 영역으로 확장시킨 독창적인 작문 보조 도구 완성
  • 100% 로컬 구동 및 RAG 제어 기능을 통해, AI에게 의도적인 '불협화음'을 주문하여 표현의 한계를 극복하는 새로운 패러다임 제시
  • 웹 UI를 통해 사용자가 AI의 작동 방식을 직관적으로 제어할 수 있는 높은 수준의 사용자 경험(UX) 구현
  • LLM의 예측 불가능성을 '버그'가 아닌 '기능'으로 제어할 수 있음을 배웠습니다.
  • Ollama를 통해 로컬 AI를 웹 서비스에 연동하는 풀스택 아키텍처를 설계하고 직접 구현해본 경험을 얻었습니다.
  • 개인적으로는 ComfyUI처럼 사용자가 직접 RAG 노드를 시각적으로 연결하는 UI(React Flow 등 활용)로 고도화하고 싶습니다.
  • 로어북을 단순 텍스트가 아닌 벡터DB(Chroma, LanceDB)와 연동하여 더 정교한 의미 기반 검색이 가능하도록 RAG 파이프라인을 개선하고 싶습니다.
